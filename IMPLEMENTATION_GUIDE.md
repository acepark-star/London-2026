# 구현 지침 — 내 노션으로 여행 웹앱 만들기

> `PRD.md`의 제품을 **본인 노션 페이지 기준으로 재현**하는 실전 가이드.
> 기본 흐름: **① 노션을 규격대로 정리 → ② 데이터 추출 → ③ `index.html`에 주입 → ④ 배포**.
> Claude(Cowork/Code) 같은 AI 에이전트에게 시키는 것을 전제로 하되, 수동으로도 가능하도록 규격을 명시한다.

---

## 0. 준비물

- 여행 계획이 정리된 **노션 페이지**(공개 공유 링크 권장 — "웹에 게시" 또는 링크 공유 ON).
- 예약 확인서(PDF/이미지) 파일들.
- (선택) 정적 호스팅 계정: GitHub Pages / Netlify / Vercel 등.
- (AI 추출을 쓸 경우) 브라우저 제어가 가능한 에이전트 + 노션 페이지를 연 브라우저.

---

## 1. 노션 페이지 규격

앱은 노션의 **표(데이터베이스)** 를 그대로 읽어 만든다. 아래 표들을 만들어 두면 추출이 쉽다. 컬럼명은 유사하면 되고, 없는 값은 비워도 된다.

### 1-1. 메인 일정 표 (필수)
| 컬럼 | 의미 | 예 |
|---|---|---|
| 날짜 | `YYYY.MM.DD(요일)` 또는 N일차 | 2026.07.11(토) |
| 시간 | `HH:MM~HH:MM` | 09:00~10:00 |
| 일정 | 항목 제목 | 아침식사 - Dishoom |
| 비고 | 메모 | 포토벨로마켓 근처 |
| 운영시간 | 장소 영업시간 | 토 09:00~24:00 |
| 이동 | 교통 문장 | 튜브 / 25분 / A → B |
| 비용 / 결제수단 | (선택) | |

> **이동 문장 규칙**(TfL 스트립 파싱용): `수단 / 소요시간 / 경로`.
> 수단 키워드: `튜브`(tube), `버스 N번`(bus), `도보`(walk), `보트`(boat), `우버`(uber).
> 예: `버스 / 37분 / 15번`, `도보 / 8분 / 550m`, `튜브 24분 또는 보트 29분 · A → B`.

### 1-2. 식당 표
`일차 · 날짜 · 구분(아침/점심/저녁/야식/애프터눈티) · 위치 · 식당명 · 메뉴 · 운영시간 · 예약필요 · 예약상태 · 예상비용`

### 1-3. 박물관·미술관 표 / 관광지(투어스팟) 표
`일차 · 날짜 · 이름 · 구분 · 비용 · 예약필요 · 예약상태 · 운영시간 · 구매처 · 비고`

### 1-4. 항공 / 숙소 (토글 또는 표)
- 항공: 항공사·상태·예약번호·비용·구간(출도착/편명/날짜).
- 숙소: 이름·기간·예약번호·비용·주소·체크인/아웃.

### 1-5. 장소 하위페이지 (선택, 상세용)
각 장소 페이지 안에 예약 정보 + "AI 여행 가이드" 섹션을 두면 상세 화면에 렌더된다.

### 1-6. 상태값 매핑
| 노션 표기 | 앱 `resv` |
|---|---|
| 예약완료 | `ok` |
| 예약필요 / 필요 | `wait` |
| 예약불필요 / 필요없음 | `free` |
| (빈칸) | `null` |

## 2. 데이터 추출

노션은 JS로 렌더되고 교차출처 제약이 있어 단순 fetch로는 안 된다. 두 가지 방법.

### 방법 A — 에이전트 브라우저 추출 (권장)
1. 노션 페이지를 브라우저에서 연다.
2. 에이전트에게 페이지 텍스트 추출을 시킨다(예: Chrome 도구 `get_page_text`).
3. 표 내용을 위 데이터 모델(`DAYS/FOOD/PLACES/...`)로 변환.
4. 접힌 토글(예: 서류·예약 상세)은 **먼저 펼친 뒤** 추출.

### 방법 B — 노션 API (보강)
- 페이지 블록 트리: `/api/v3/loadPageChunk`, 파일 URL: `/api/v3/getSignedFileUrls` (노션 세션에서 호출).
- 대용량은 배치로 나눠 호출(타임아웃 주의).

### 이미지 처리 주의
- 노션 이미지 프록시 URL(`.../image/...`)은 `<img src>`로 **표시**는 되지만 **다운로드(fetch)는 CORS로 막힘**.
- 따라서 예약 스크린샷·티켓 이미지는 **노션에서 직접 저장해 `vouchers/` 폴더에 파일로 넣는 것**을 표준으로 한다(아래 4장).

## 3. `index.html`에 데이터 주입

`index.html`의 `<script>` 안 데이터 배열을 채운다. (구조는 `PRD.md` 6장)

```js
const DAYS = [
  { n:1, date:"07.10", dow:"금", theme:"런던 입성", events:[
    {t:"10:55–17:20", k:"flight", ttl:"인천 → 런던 KE907", note:"대한항공"},
    {t:"18:30–20:00", k:"move", ttl:"런던 시내 이동", move:"튜브 46분 · Heathrow → Charing Cross"},
    {t:"20:30–21:30", k:"food", ttl:"마트 — Tesco Express", map:"Tesco Express Strand London"},
    {t:"~", k:"rest", ttl:"숙소 휴식"},
  ]},
  // ... 날짜별 반복
];

const FOOD = [
  {day:2, date:"07.11(토)", slot:"아침", icon:"🍽️", area:"노팅힐",
   name:"Dishoom Permit Room", menu:"인도 음식점", hour:"09:00~24:00 (토)", resv:null, map:"Dishoom Permit Room London"},
];

const PLACES = [
  {day:4, date:"07.13(월)", cat:"박물관·미술관", type:"🏛️", name:"웨스트민스터 사원",
   price:"£31", time:"12:30 입장", hour:"월~금 09:30~15:30", resv:"ok", via:"마이리얼트립", note:"", map:"Westminster Abbey London"},
];

const VOUCHERS = {
  "웨스트민스터 사원": [["입장권", "voucher_westminster.pdf"]],
  // 이름은 FOOD/PLACES의 name과 정확히 일치해야 매칭됨
};

const DOCS = [
  {group:"✈️ 항공", items:[{label:"전자항공권 (홍길동)", file:"ETKT_HONG.pdf"}]},
  {group:"📄 기타 서류", items:[{label:"여행자보험", file:"insurance.pdf"}, {label:"eSIM", todo:true}]},
];
```

### 규칙
- `PLACE_PAGES` / `FOOD_PAGES` 배열은 **PLACES / FOOD 와 인덱스가 1:1**로 맞아야 한다. 항목을 중간에 추가·삭제·재정렬하면 두 배열을 **함께** 수정하고, 페이지가 없으면 `""`(빈 문자열)로 자리를 채운다.
- `vurl(fn)`은 `vouchers/` + `encodeURIComponent(fn)` 이므로 파일명에 공백이 있어도 되지만, **공백 없는 ASCII 파일명**을 권장.
- 지도 링크는 `map:"검색어 London"` 형태(구글맵 검색). 주소 URL이 있으면 `mapHref` 사용.

## 4. 첨부 파일 (`vouchers/` 폴더)

`index.html`과 **같은 위치**에 `vouchers/` 폴더를 두고 예약 문서를 넣는다.

- 파일명은 `VOUCHERS` / `DOCS` / `FLIGHT.vouchers` / `ACCOM.vouchers`에 적은 이름과 **정확히 일치**.
- 종류: PDF·PNG·JPG 모두 가능(상세/서류에서 새 탭으로 열림).
- **갱신 방법**: 내용이 바뀌면 **같은 파일명으로 덮어쓰기**만 하면 코드 수정 불필요.
- 노션에만 있는 이미지(스크린샷 등)는 노션에서 다운로드해 이 폴더에 저장 후 파일명 등록.

권장 파일명 예: `voucher_<place>.pdf`, `ticket_<place>.pdf`, `ETKT_<NAME>.pdf`, `GWR_1_LON-BTH_1.pdf`.

---

## 5. 아이콘 / 외형

- `favicon-192.png`(192×192, 탭·안드로이드), `apple-touch-icon.png`(180×180, iOS 홈 화면)을 `index.html` 옆에 둔다. `<head>`에 링크되어 있음.
- 정사각형 이미지를 넣으면 됨(모서리 라운딩은 OS가 처리).
- (선택) 헤더 배경: `header.jpg`가 있으면 자동 반영, 없으면 단색.
- 테마색은 `#1d1d1f`로 고정(iOS 상태바 통일).

---

## 6. 배포

1. 아래 파일들을 **같은 폴더**에 둔다.
   ```
   index.html
   favicon-192.png
   apple-touch-icon.png
   header.jpg            (선택)
   vouchers/            (예약 문서 전체)
   ```
2. GitHub 저장소에 올리고 **Settings → Pages**에서 배포(또는 Netlify에 폴더 드래그).
3. 발급된 URL을 휴대폰에서 열고 **홈 화면에 추가**.
4. 아이콘/데이터가 안 바뀌면 브라우저 **강력 새로고침**(캐시).

---

## 7. 노션 변경 시 갱신 루틴

1. 노션에서 일정/식당/장소를 수정.
2. 데이터 재추출(2장) → `index.html` 배열 갱신(3장).
3. 문서가 바뀌면 `vouchers/`의 파일을 같은 이름으로 교체(4장).
4. 배포 폴더에 덮어쓰기 → 강력 새로고침.

> 팁: 표 항목 순서를 바꾸면 `PLACE_PAGES/FOOD_PAGES` 정렬이 깨질 수 있으니, 변경 후 **개수·매핑을 검증**한다(아래 8장).

---

## 8. 검증 체크리스트

- [ ] `FOOD.length === FOOD_PAGES.length`, `PLACES.length === PLACE_PAGES.length`.
- [ ] 대표 항목 몇 개의 `name → 페이지 슬러그` 매핑이 맞는지 확인.
- [ ] `<script>`가 문법 오류 없이 실행되는지(브라우저 콘솔 또는 Node 파싱).
- [ ] 모바일 폭에서 가로 스크롤이 생기지 않는지.
- [ ] now-line이 현지 시간대(`Intl` timeZone) 기준으로 뜨는지.
- [ ] 모든 [열기]/바우처 버튼이 실제 파일을 여는지(파일명 오타·대소문자).

---

## 9. AI 에이전트에게 시킬 때 (예시 프롬프트)

```
이 노션 페이지( <내 노션 URL> )의 여행 일정을 휴대폰용 단일 HTML 웹앱으로 만들어줘.
- PRD.md / IMPLEMENTATION_GUIDE.md 규격을 따를 것
- 하단 탭: 일정 / 식당 / 장소 / 항공·숙소 / 서류
- 일정은 연속 스크롤 + sticky 날짜칩·헤더 + 현지시간 now-line
- 이동 구간은 TfL 스타일 교통 스트립
- 예약 문서는 vouchers/ 폴더의 로컬 파일로 연결(서류 허브)
- Apple 디자인 토큰(위 표) 적용, 테마색 #1d1d1f
먼저 노션에서 표를 추출해 데이터 모델로 변환하고, index.html에 주입한 뒤,
배열 길이·매핑을 검증하고 결과를 보여줘.
```
