# PRD — 노션 연동 여행 웹앱 (Travel Itinerary Web App)

> 노션에 정리한 여행 일정을 **휴대폰에서 빠르게 보는 단일 HTML 웹앱**으로 만드는 프로젝트.
> 이 문서는 다른 사람이 **자신의 노션 페이지로 똑같이 만들 수 있도록** 제품 요구사항을 정의한다.
> 실제 제작 절차는 함께 있는 `IMPLEMENTATION_GUIDE.md`를 참고.

---

## 1. 개요

| 항목 | 내용 |
|---|---|
| 제품명 | 여행 일정 웹앱 (예: "런던 여행") |
| 형태 | 백엔드 없는 **정적 단일 파일** `index.html` (HTML+CSS+JS 인라인) |
| 배포 | GitHub Pages 등 정적 호스팅, 또는 파일 직접 열기 |
| 사용 환경 | 모바일 우선 (세로 화면), iOS/안드로이드 브라우저 · 홈 화면 추가 |
| 데이터 원천 | 개인 **노션 페이지**(여행 계획)에서 추출해 앱 안에 정적 데이터로 주입 |
| 목적 | 여행 중 일정·예약·서류를 오프라인에서도 즉시 확인 |

### 핵심 원칙
- **한 파일로 완결**: CSS/JS를 분리하지 않고 `index.html` 하나에 인라인. (첨부 폴더 `vouchers/`, 아이콘 PNG는 예외)
- **정적·오프라인 우선**: 예약 바우처·티켓 등은 파일로 동봉해 노션 상태와 무관하게 열림.
- **노션은 원천, 앱은 스냅샷**: 노션이 바뀌면 데이터를 다시 추출해 앱을 갱신한다(자동 동기화 아님).

---

## 2. 대상 사용자 / 시나리오

- 여행 계획을 노션에 정리해 둔 개인 여행자.
- 여행 중 휴대폰으로 "지금 뭐 하지 / 예약 됐나 / 티켓 어디 있지 / 어떻게 가지"를 빠르게 확인.
- 대표 시나리오
  - 아침에 앱을 열면 **현재 현지 시각 기준 위치**가 타임라인에 표시된다.
  - 식당 입구에서 [식당] 탭 → 예약 바우처를 바로 연다.
  - 입국심사·기차 탑승 때 [서류] 탭에서 티켓을 즉시 연다.
  - 이동 구간에서 교통수단·소요시간·지도 링크를 확인한다.

---

## 3. 정보 구조 (탭)

하단 고정 탭 5개 + 상세 화면.

1. **일정 (Timeline)** — 전체 여행을 하나의 연속 스크롤로. 날짜가 바뀌는 지점에 대제목, 상단에 날짜 칩 네비게이터.
2. **식당 (Food)** — 날짜별 식사(아침/점심/저녁/야식/애프터눈티) 카드.
3. **장소 (Places)** — 박물관·미술관·공연·투어 스팟을 날짜별로.
4. **항공·숙소 (Book)** — 항공편/숙소 예약 정보, 예약번호·비용·바우처.
5. **서류 (Docs)** — 모든 예약 문서를 한 곳에 모은 얇은 인덱스(이름 + 열기).

상세 화면(Detail): 식당/장소 카드를 탭하면 예약 정보 + "AI 여행 가이드" 등 노션 하위페이지 내용을 네이티브로 렌더.

## 4. 기능 요구사항

### 4.1 일정 타임라인
- 전체 일정을 **끊김 없는 하나의 스크롤**로 표시.
- **Sticky 날짜 칩 바**: 상단에 `1일·2일·…` 칩. 탭하면 해당 날짜로 스크롤. 스크롤 위치에 따라 현재 날짜 칩 하이라이트(스크롤스파이).
- **Sticky 날짜 헤더**: 각 날짜 대제목이 스크롤 시 상단에 붙었다가 다음 날짜에서 교체(sticky stacking).
- **현재 시각 표시선(now-line)**: 실제 **현지 시각**(예: 런던 `Europe/London`)을 기준으로, 진행 중인 일정 위치에 깜빡이는 실선. 실시간 갱신.
- 이벤트 종류(k): `flight/see/food/shop/move/rest`. 종류별 색 점(dot)·이모지.
- **이동(move) 항목**: 시간과 내용을 한 줄로, 그 아래 **TfL(런던 교통공사) 스타일 교통 스트립**(튜브/버스/도보/보트 + 소요시간 + 역 경로 + 지도 링크).
- 각 항목: 시간 → 제목(이모지) → 비고(픽토그램) → 운영시간(픽토그램) → 하위목록 → 지도 링크.
- 예약 상태 뱃지(예약완료/예약필요/예약불필요)는 **제목 아래 줄**에 배치.

### 4.2 식당 / 장소 카드
- 카드 구성(위→아래): 슬롯 뱃지(아침/점심/…) → 제목 → **예약완료·바우처 뱃지(제목 아래 줄)** → 위치·메뉴 메타 → **운영시간(별도 줄)**.
- 메타는 라인 픽토그램(위치/메뉴/시간을 선 아이콘)으로 통일.
- 예약 바우처가 있으면 "바우처" 칩 표시.

### 4.3 상세 화면 (Detail)
- 예약 정보(사이트/금액/예약번호/취소규정)와 "AI 여행 가이드" 본문을 노션에서 추출해 렌더.
- 상단 "뒤로" + **인터랙티브 백 스와이프**(왼쪽 가장자리에서 밀기).
- 바우처 버튼 → 파일을 새 탭에서 열기.

### 4.4 항공·숙소
- 항공: 항공사·예약번호·비용·구간(leg)·바우처(전자항공권).
- 숙소: 숙소별 기간·예약번호·비용·주소(지도)·체크인/아웃·바우처.

### 4.5 서류 허브 (얇은 인덱스)
- 모든 문서를 그룹(항공/숙소/기차/식당/입장권·공연/기타)으로 모아 **이름 + [열기] 버튼**만 노출.
- 파일이 아직 없는 항목은 "추가 예정" 상태로 자리만.
- 파일 방식(로컬 `vouchers/`)을 표준으로 함 → 갱신은 "같은 파일명으로 덮어쓰기".

### 4.6 지도 링크
- 장소명 기반 구글맵 검색 URL(`https://www.google.com/maps/search/?api=1&query=...`).
- 주소가 있으면 구글맵 단축 URL 직접 사용.

### 4.7 외형 / 플랫폼
- 최상단 헤더에 배경 이미지 옵션(`header.jpg` 있으면 반영, 없으면 단색).
- **파비콘/애플 터치 아이콘**(`favicon-192.png`, `apple-touch-icon.png`).
- iOS 상태바/테마색 통일(`theme-color` = `#1d1d1f`), 홈 화면 추가 대응.

## 6. 데이터 모델 (앱 내 정적 배열)

> 노션에서 추출한 값을 아래 구조로 `index.html` 스크립트에 넣는다.

- `DAYS[]` — 일정. `{n, date:"MM.DD", dow, theme, events:[{t, k, ttl, note?, hour?, resv?, move?, map?, subs?}]}`
- `FOOD[]` — `{day, date, slot, icon, area, name, menu, hour, resv, map}`
- `PLACES[]` — `{day, date, cat, type(이모지), name, price, time, hour, resv, via, note, map}`
- `FLIGHT{}` — `{airline, status, resNo, cost, vouchers:[[label,file]], legs:[...]}`
- `ACCOM[]` — `{name, tag, resNo, cost, area, checkin, checkout, map|mapHref, vouchers}`
- `VOUCHERS{}` — `{ "정확한 이름": [[label, filename], ...] }` (이름으로 FOOD/PLACES에 매칭)
- `DOCS[]` — 서류 허브. `{group, items:[{label, file?, note?, todo?}]}`
- `NOTION_CONTENT{}` — 상세용 노션 본문. `{ pageId: [[blockType, text, href?], ...] }`
- `PLACE_PAGES[]` / `FOOD_PAGES[]` — **PLACES/FOOD와 인덱스 정렬**된 노션 페이지 슬러그. 배열 순서를 바꾸면 반드시 이 배열도 함께 맞출 것.
- 헬퍼: `vurl(fn) = "vouchers/" + encodeURIComponent(fn)`

`resv` 값: `"ok"`(예약완료) · `"wait"`(예약필요) · `"free"`(예약불필요) · `null`.
`k` 값: `flight · see · food · shop · move · rest`.

## 7. 기술 제약 / 결정

- **iframe 임베드 불가**: 노션은 교차출처 iframe·fetch(CORS)를 막음 → 노션 내용은 **추출 후 네이티브 렌더**.
- **노션 이미지**: 프록시 URL(`/image/...`)은 `<img>` 표시(교차출처)는 되지만 **다운로드(fetch)는 CORS로 막힘**. 서명된 파일 URL은 만료됨.
- **바우처/문서**: 노션 링크 의존 대신 **로컬 파일 방식** 채택(오프라인·영구·갱신 용이).
- **단일 파일** 유지, 외부 스크립트는 필요 시 cdnjs만.

## 8. 비목표 (Non-goals)

- 실시간 노션 동기화(양방향 연동) 없음.
- 로그인·다중 사용자·서버 저장 없음.
- 지도 자체 임베드(구글맵 API) 없음 — 외부 링크로 대체.
- 체크리스트(준비물/할일) 탭 없음 — 여행 중 효용이 낮아 서류 허브로 대체.

## 9. 완료 기준 (Acceptance)

- [ ] 휴대폰에서 좌우 스크롤(가로 오버플로) 없이 화면에 꽉 참.
- [ ] 일정 탭이 연속 스크롤 + sticky 날짜 칩/헤더 + 실시간 now-line으로 동작.
- [ ] 식당/장소/항공·숙소/서류 탭이 노션 데이터와 일치.
- [ ] 모든 예약 문서가 서류 허브에서 [열기]로 열림(로컬 파일).
- [ ] 파비콘·테마색·홈 화면 아이콘 반영.
- [ ] 노션 변경 시 데이터 재추출로 갱신 가능(가이드 절차대로).
